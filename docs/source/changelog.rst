### [RIS-2563](https://radpoint.atlassian.net/browse/RIS-2563) \[RPT\] "Rodzaje zlecen" - grid filter modification

##### Description

Jako lekarz chcę domyślnie  widzieć zlecenia opisów zarówno lokalnych jak i z teleradiologii 

gdzie \(definicje\):

**Opisy lokalne \(Lokalne\)** - zlecenia wysyłane do RPT bez kontekstu umowy teleradiologicznej , zdecydowana większość zleceń opisu generowana w zwykłej pracowni lokalnie u klienta RIS/PACS, który nie zleca do opisu w teleradiologii i nie świadczy usług teleradiologii na rzecz innych podmiotów \(np. jako lekarz pracujący w Ostrołęce w lokalnej pracowni TK widzę badania przesłane do opisu z tej pracowni TK - technik wysłał do opisu lokalnego\)

**Zlecenia teleradiologiczne \(Teleradiologia\)** -  czyli zlecenia świadczone pomiędzy różnymi podmiotami, przesyłane do RPT przez zewnętrzne podmioty zlecające \(innych klientów\). Czyli jesli np. z sochaczewa wyśle technik badanie do opisu do voxela, to będzie to zlecenie widoczne w RPT voxela jako zlecenie teleradiologiczne.

| **Label przycisku “Rodzaj zleceń”** | `serviceType` | `provider` | `recipient` |
| --- | --- | --- | --- |
| Lokalne | null | null | null |
| Lokalne i teleradiologia | `TELERADIOLOGY` lub null | `provider.id` = klient id LUB null | `recipient.id` **!=** klient id LUB null |
| Teleradiologia | `TELERADIOLOGY` | `provider.id` = klient id  | `recipient.id` **!=** klient id |
| XXX - gdzie XXX odpowiada `serviceName` umowy z API | `TELERADIOLOGY` | `provider.id` = klient id | `recipient.id` = klient id  |

### [RIS-2596](https://radpoint.atlassian.net/browse/RIS-2596) \[DOSE\] Add "Wypełnij automatycznie" button

##### Description

Jako technik/rejestratorka chcę móc wypełnić automatycznie “na życzenie” dane dotyczące dawki. 

Obecnie system uzupełnia dane dotyczące dawki automatycznie \(w oparciu o powiązany ze zleceniem numer StudyUID\) tylko jeśli zlecenie w REG:  
- nie ma wprowadzonych żadnych wprowadzonych danych w sekcji “Dawki i ekspozycje”  
- klient ma aktywną licencję na Dawki \(od tego zależy czy widzi tą sekcję\)  
- performer którego dotyczy zlecenie ma włączony monitoring dawek dla TK lub RTG  
- badanie zeszło na PACS i zostało przeanalizowane przez algorytm backendowy dotyczący dawki   
- badanie które jest powiązane ze zleceniem zostało wykonane na obsługiwanym przez algorytm aparacie   
  
Może się zdarzyć, że użytkownik pomimo tego, iż wprowadził dane dt. dawek ręcznie chce mimo wszystko dopisać dane będące wynikiem analizy algorytmu AI \(np. dla TK - parametr SSDE użytkownik nie jest w stanie sam wyliczyć, albo dla RTG nie jest w stanie wyczytać z DICOM bezpośrednio nastawów aparatu\) 

Prośba o dodanie przycisku w oknach dt. wprowadzania dawek który po kliknięciu automatycznie sprawdzi dane w PRS dt. wyników AI\_DOSE\_CR lub \_CT i uzupełni dane w formularzu. Dane **mają być dodawane** jako osobne pozycje do ewentualnie już istniejących. 

Jeśli w PSR brak danych to powinna być informacja zwrotna dla użytkownika, iż brak wyników analizy dawek dla powiązanego ze zleceniem badania.
